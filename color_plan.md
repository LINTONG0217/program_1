# 颜色识别方案

## 1. 推荐目标定义

- 物体：高饱和红色或绿色圆柱/方块
- 指定区域：蓝色或黄色矩形框/地贴
- 场地边界：10cm 黄色胶带（可作为全局定位地标）
- 避免背景与目标颜色接近

## 2. 现场标定步骤

1. 固定赛场光照
2. 在 OpenMV IDE 或 K210 调试界面查看 LAB 数值
3. 分别采样物体和区域颜色
4. 先收紧阈值，再逐步放宽
5. 检查不同距离下是否仍能稳定识别

## 3. 阈值建议

OpenMV/K210 常用 `find_blobs()` 的 LAB 阈值格式：

`(L_min, L_max, A_min, A_max, B_min, B_max)`

示例：

- 红色物体：`(25, 75, 15, 70, 15, 70)`
- 蓝色区域：`(20, 80, -70, -10, 10, 70)`
- 绿色物体：`(20, 70, -70, -20, 10, 70)`
- 黄色区域：`(40, 90, -10, 20, 40, 90)`
- 黄色胶带边界（示例）：`(40, 95, -15, 25, 35, 95)`（以现场标定为准）

## 4. 识别策略

- 物体：取最大色块
- 目标区：取最大矩形区域
- 设置 `pixels_threshold` 与 `area_threshold`，过滤噪声
- 开启 `merge=True`，减少碎块分裂
- 关闭自动白平衡和自动增益，避免颜色漂移

## 5. 赛场建议

- 物体颜色与地面反差明显
- 目标区周围留空白边界
- 避免透明、反光、金属材质
- 摄像头俯角固定，不要在运行中晃动

## 开机全局定位（主车）

你现在的工程里，里程计/融合默认是“开机把当前位置当原点”。如果你希望一开机就直接得到场地坐标系下的位姿(全局定位)，需要在开机阶段把里程计坐标系对齐到场地坐标。

## A. 起点固定时（推荐先做）

如果主车每次都从固定的起点、固定朝向出发（比如左下角附近，面向+x方向），那么开机全局定位可以直接用配置注入初始位姿实现。

在 config.py 打开：

- POSE_INIT_ENABLE = True
- POSE_INIT_X / POSE_INIT_Y：主车开机位置（米）
- POSE_INIT_YAW_DEG：主车开机朝向（度），定义为朝向+x方向时为0度

坐标系建议：

- 场地左下角为(0,0)
- x沿3.2m方向，y沿2.4m方向

这样启动后第一帧IMU航向出来时，系统会把里程计的(x,y,yaw)一次性对齐到你的场地世界坐标。

## B. 起点不固定时（真正“自动全局定位”需要地标）

如果主车可能从场地任意位置/任意朝向开机，仅靠轮速+IMU无法唯一确定(x,y)，需要外部可观测的地标：

- 最简单：在场地四角贴4个不同颜色角标（红/绿/黄/紫等，背景是蓝布很利于分割）
- 你现在的场地边界是10cm黄色胶带：可以把“边界矩形/角点”当作天然地标（前提：摄像头开机能看到足够多的边界，最好能看到一个角或整条边）
- 或者：AprilTag/ArUco 等二维码标记（更稳，但需要对应视觉算法支持）

实现思路：

1) 摄像头开机拍到≥2个角标（越多越好）
2) 由角标像素坐标 + 已知场地角点世界坐标，解算相机/车体在场地中的(x,y,yaw)
3) 把解算结果写入 POSE_INIT_* 或直接注入里程计初始化

### 已接入：俯视 OpenArt + 单角点开机定位

你当前条件（俯视、OpenArt、开机能看到一个黄色边界角）已经接入：

- `vision_openart.py` 会输出 `field_corner`
- `main_openart.py` 开机会尝试从 `field_corner` 解算位姿并注入估计器

需要在 `config.py` 标定这些参数：

- `OPENART_VISIBLE_WORLD_CORNER`：可见角是 BL/BR/TL/TR 哪一个
- `OPENART_PIXEL_TO_M_X`、`OPENART_PIXEL_TO_M_Y`：像素到米
- `OPENART_PIXEL_TO_ROBOT_X_SIGN`、`OPENART_PIXEL_TO_ROBOT_Y_SIGN`：像素轴方向到机器人轴方向符号
- `OPENART_CAM_TO_ROBOT_X_M`、`OPENART_CAM_TO_ROBOT_Y_M`：相机中心到机器人参考点偏移

建议先跑 `main_openart_debug.py`，观察 `field` 和 `field_corner` 输出是否稳定，再做像素到米标定。
